# SPP 弦粒子逆建模 Demo V2 — 实施方案

## 一、目标概述

基于 `spp-lib`（`spp-core.js` + `spp-inverse-engine.js`），实现全新的弦粒子逆重建 Demo：

1. **全屏 Canvas**：Three.js 铺满视口，所有交互在 3D 场景内完成
2. **多深度可视化**：粗网格 → 细化层叠加覆盖，按 depth 分层分色
3. **AI 自主递归细化**：Phase 5 顶视图回归对比，AI 自动识别偏差区域并细化；用户手动框选为补救手段
4. **房型图贴地板**：导入图片作为 3D 地面纹理，直观对比契合度

```

  ↓

  ↓

  ↓

  ↓

  → 对每块偏差区域，重新走一遍局部 Phase 1-4：

      局部 Phase 3: AI 重新识别该区域门窗

  → 重复直到无显著偏差或达到 depth 上限
  ↓
用户如不满意 → 手动框选特定区域 → 触发同一细化流程（补救手段）
  ↓
快捷键 E → 导出 SPP JSON
```

## 四、核心模块设计

### 4.1 多层渲染器 (`renderer.js`)

按 `depth` 分组渲染，每层不同颜色 + 微微抬高：

| depth | 墙体颜色 | 地板颜色 | 墙高 | 抬高 Y |
|-------|---------|---------|------|--------|
| 0（粗网格） | `#d4886b` 砖红 | `#e8e8ef` 浅灰 | 2.8 | 0 |
| 1（一次细化）| `#6b8fd4` 蓝灰 | `#e0e8f0` 淡蓝 | 2.6 | 0.08 |
| 2（二次细化）| `#6bd49b` 青绿 | `#e0f0e8` 淡绿 | 2.4 | 0.16 |

被细化区域覆盖的粗格子 → 墙体变半透明（`opacity: 0.15`），地板隐藏。

### 4.2 顶视图回归模块 (`regression.js`)

Phase 5 的核心，负责自动驱动细化循环：

```
renderTopView()          ← 切换正交相机，截取当前 Canvas 为图像 A
compareWithOriginal(A, B) ← 按 cell 粒度做像素差分，返回偏差 cell 列表
cropRegion(B, cells)     ← 从原始平面图 B 中裁剪偏差区域小图
extractBoundaryConstraints(cells) ← 提取选区四边的 Open/Wall 连通状态
```

**偏差判断标准：** 按 cell 对应的像素块做均值颜色差，超过阈值（可配置）则标记为偏差。

**截图实现：** Three.js `renderer.render()` 后调用 `canvas.toDataURL()`，临时切换为正交相机（保持场景不变）。

### 4.3 手动多选机制 (`selection.js`)

作为 Phase 5 自动细化的**补救手段**，在用户对自动结果不满意时使用：

- **单击 cell**：选中/取消选中
- **Ctrl+Click**：追加多选
- **左键拖拽**：框选
- **选中高亮**：蓝色描边 + 地板微微变亮
- 选中后底部出现 `"🔍 细化选中区域 (N cells)"` 按钮

#### 4.3.1 框选与 OrbitControls 的鼠标分工

| 操作 | 触发条件 | 行为 |
|------|---------|------|
| 场景旋转 | 右键拖拽 | OrbitControls rotate |
| 场景平移 | 中键拖拽 或 右键+Shift | OrbitControls pan |
| 框选 | 左键拖拽（无修饰键）| 绘制选区矩形 |
| 单选/取消 | 左键单击（mousedown→mouseup 无位移）| 选中/取消该 cell |

实现：`controls.mouseButtons = { LEFT: null, MIDDLE: THREE.MOUSE.PAN, RIGHT: THREE.MOUSE.ROTATE }`，左键完全由 `selection.js` 接管。

### 4.4 房型图贴地板 (`floor-texture.js`)

- 用 `TextureLoader` 加载图片，UV 偏移到 crop 区域
- 铺在 `y=0.005`（低于所有网格地板）
- 透明度通过右下角滑块实时调节（0%~100%）

### 4.5 快捷键

| 快捷键 | 功能 |
|--------|------|
| `T` | 俯视 / 透视切换 |
| `E` | 导出 SPP JSON（递归树结构） |
| `Escape` | 清空选择 |
| `Delete` | 移除选中区域的细化层（回退到上一层） |

## 五、完整重建流程（Phase 1–5）

### 5.1 Phase 1：粗网格识别

**AI 任务：** 检测平面图边界 + 覆盖网格 + 分配语义房间名。

**输出：**
```json
{
  "crop": { "x": 0.08, "y": 0.10, "w": 0.84, "h": 0.82 },
  "gridX": 5, "gridZ": 4,
  "layout": [
    ["Kitchen", "Kitchen", "Hallway", "Bathroom", "Bathroom"],
    ["Living Room", "Living Room", "Hallway", "Bedroom", "Bedroom"]
  ]
}
```

### 5.2 Phase 2：二值拓扑生成

**AI 任务：** 对每个 cell 的四条水平边分类：同房间 → `[0]` Open，不同房间或外墙 → `[10]` Wall。不生成门窗。

### 5.3 Phase 3：门窗位置识别

**AI 任务：** 给定原始图 + Phase 2 的 wall face 列表，返回门窗标注数组 `[{ x, z, face, optionId }]`。

optionId 取值：`1`（拱门）、`2`（方门）、`20`（窗）。规则：只标注当前为 Wall 的面，门需标注两侧 cell。

### 5.4 Phase 4：特征穿透（确定性）

遍历 Phase 3 标注，把 `faceOptions[face]=[10]` 替换为对应 optionId。只替换 Wall(10) 面，不触碰 Open(0) 面。

### 5.5 Phase 5：顶视图回归对比（局部 Phase 1-4 递归闭环）

Phase 5 不是一个独立的"细化"步骤，而是**驱动局部 Phase 1-4 反复执行直到收敛**的闭环控制器。

**全局首次 vs 局部递归的区别：**

| | 全局首次（Phase 1-4） | 局部递归（Phase 5 触发） |
|--|--|--|
| Phase 1 输入 | 完整平面图 | crop 出的偏差区域小图 |
| Phase 1 约束 | 无 | 四边 Open/Wall 连通约束 |
| Phase 2-4 | 同上 | 同上，仅针对该局部 |
| 结果整合 | 建立主网格 | integrateRegionSubGrid() 写回主网格对应区域 |

**执行流程：**

```
① 渲染正交顶视图截图（图像 A）
   ↓
② 前端像素对比：原始平面图（图像 B）vs 图像 A
   → 按 cell 粒度采样，找出偏差超阈值的 cell 集合
   ↓
③ 对每个偏差区域（相邻偏差 cell 合并为一块），执行局部 Phase 1-4：
   局部 Phase 1: crop 小图 → AI 重新划局部网格（房间边界 + 语义名）
   局部 Phase 2: AI 重新生成该区域二值拓扑
   局部 Phase 3: AI 重新识别该区域门窗位置
   局部 Phase 4: 确定性穿透写入 faceOptions
   → integrateRegionSubGrid() 写回主网格
   ↓
④ 重新渲染，回到 ① 继续比较
   → 无新偏差 或 达到 depth 上限 → 停止
```

**自动细化完成后**，用户可手动框选仍不满意的区域触发额外细化。

### 5.6 完整调用链

```
── 全局首次重建 ──────────────────────────────────────────
Phase 1: analyzeGridSize(imageDataUrl)                  → gridInfo
Phase 2: classifyFaces(imageDataUrl, gridInfo)          → { cells }
Phase 3: detectFeatures(imageDataUrl, cells, gridInfo)  → annotations[]
Phase 4: pierceFeatures(cells, annotations)             → finalCells
render Layer 0 (depth=0)

── Phase 5 自动闭环（局部 Phase 1-4 递归）────────────────
loop until converged or depth limit:
  imageA = renderTopView()
  diffRegions = compareWithOriginal(imageA, imageB)
  if diffRegions is empty → break

  for region of diffRegions:
    cropImage = cropRegion(imageB, region)
    constraints = extractBoundaryConstraints(region)

    // 局部 Phase 1-4（输入换成小图+约束）
    localGridInfo  = analyzeGridSize(cropImage, constraints)
    localCells     = classifyFaces(cropImage, localGridInfo)
    localAnn       = detectFeatures(cropImage, localCells, localGridInfo)
    localFinal     = pierceFeatures(localCells, localAnn)

    integrateRegionSubGrid(region, localFinal)

  re-render

── 用户手动补救（同一局部 Phase 1-4）──────────────────────
on user selects region manually:
  触发与上方 for region 相同的局部 Phase 1-4 流程
```

## 六、细化网格与主网格的关系

### 6.1 区域级 subGrid（非 per-cell）

细化操作以**选区整体**为单位，而非逐 cell 独立细化。

选中 2×2 区域，scale=3 → 生成**一个统一的 6×6 子网格**，覆盖整个选区的世界坐标范围：

```
主网格（2×2）              细化后
┌───┬───┐                ┌─┬─┬─┬─┬─┬─┐
│   │   │   → scale=3 →  │ │ │ │ │ │ │
├───┼───┤                ├─┼─┼─┼─┼─┼─┤
│   │   │                │ │ │ │ │ │ │  (6×6 统一子网格)
└───┴───┘                └─┴─┴─┴─┴─┴─┘
```

6×6 子网格在世界坐标中占据与原 2×2 完全相同的空间范围。原 2×2 cells 渲染变半透明。

### 6.2 边界连通约束（仅 Open/Wall）

子网格的约束条件极简：**只需保持选区四条外边的连通状态一致**，不要求精确对应到哪条具体 face。

```
原 2×2 区域左边有 Open  → 6×6 子网格左列中至少有一个 Open
原 2×2 区域右边全是 Wall → 6×6 子网格右列全是 Wall
内部墙体走向完全由 AI 自由决定
```

**约束提取：** 对选区四条外边，统计 Open/Wall 占比，转化为 `{ left: 'open'|'wall', right, top, bottom }`。

### 6.3 数据结构

```js
// 区域级 refinement 挂载在 RegionRefinement 对象上，而非 parentCell.subGrid
refinement = {
    region: { minX, maxX, minZ, maxZ },  // 原选区范围（主网格坐标）
    scale: 3,
    depth: 1,
    subGrid: {
        gridX: 6, gridZ: 6,
        cells: [...]      // 统一的 6×6 ParticleCell 数组
    }
}
```

主数据结构维护一个 `refinements[]` 列表，渲染时按 depth 叠加。

### 6.4 坐标映射

子网格 cell 坐标 `(sx, sz)` 到世界坐标：

```
worldX = regionMinX * cellSize + (sx / subGridX) * regionWidth
worldZ = regionMinZ * cellSize + (sz / subGridZ) * regionHeight
worldScale = regionWidth / subGridX
```

## 七、细化 AI 调用策略

### 7.1 输入：局部 crop + 连通约束

**不发送完整平面图**，只发送偏差区域的小图 + 四边连通状态，大幅节省 token：

```js
{
  image: cropDataUrl,          // 从原始平面图 crop 出的局部小图
  constraints: {
    left:   'open',            // 选区左边有 Open 连接
    right:  'wall',            // 选区右边全是 Wall
    top:    'wall',
    bottom: 'open',
  },
  rooms: ['Bedroom', 'Bathroom'],  // 选区内涉及的房间名
}
```

### 7.2 连通约束提取规则

对选区每条外边，扫描对应 cells 的外边 faceOptions：
- 若存在任意一个 Open(0) face → 该边标记为 `'open'`
- 全部为 Wall(10)/Door(1/2)/Window(20) → 该边标记为 `'wall'`

### 7.3 AI Prompt 结构（细化专用）

```
You are an SPP spatial refiner. Given a cropped floor plan region and boundary
connectivity constraints, generate a fine-grained sub-grid.

## Boundary Constraints
left: open   (this edge connects to an adjacent space)
right: wall  (this edge is a solid exterior or room boundary)
top: wall
bottom: open

## Rooms in this region
Bedroom, Bathroom

## Task
Choose a scale (2, 3, or 4) based on visual complexity.
Generate a unified NxM sub-grid where N = cols * scale, M = rows * scale.
Internal wall placement is free — only boundary connectivity must be respected.

## Output
{
  "scale": 3,
  "gridX": 6,
  "gridZ": 6,
  "cells": [{ "position": [x, 0, z], "room": "...", "faceOptions": [...] }]
}
```

### 7.4 Scale 策略

AI 根据 crop 图的视觉复杂度自主选择 `scale ∈ {2, 3, 4}`，并在输出 JSON 中返回。

## 八、Delete 回退机制

### 8.1 删除粒度：整个区域 refinement

Delete 操作以**区域 refinement 为单位**：选中某细化区域中的任意 cell → 找到对应的 `refinement` 对象 → 整体删除，原区域内的 cells 恢复显示。

### 8.2 逐层回退约束

```
用户选中 depth=1 区域中的某些 cell
  ↓
检查该 refinement 区域内是否存在 depth=2 的子 refinement
  → 若存在：Toast 提示"请先删除更深层的细化"，阻止操作
  → 若不存在：删除该 refinement，主网格对应区域恢复不透明，重新渲染
```

## 十一、已确认的设计决策

| # | 决策项 | 结论 |
|---|--------|------|
| 1 | **Scale 策略** | AI 根据区域复杂度自主选择，取值 `∈ {2, 3, 4}` |
| 2 | **递归深度上限** | 最多 4 层（depth 0~3），超过后禁用细化 |
| 3 | **导出格式** | 递归树结构 JSON，保留 refinements 嵌套层级 |
| 4 | **Delete 粒度** | 删除整个区域 refinement，不支持局部回退 |
| 5 | **细化主导** | AI 通过顶视图回归自动驱动；用户手动框选为补救手段 |
| 6 | **subGrid 粒度** | 区域级（一块选区 = 一个统一子网格），非 per-cell |
| 7 | **边界约束** | 仅 Open/Wall 连通状态（四边各一个值），不要求精确 face 匹配 |
| 8 | **细化 token 策略** | 前端 crop 局部小图 + 提取约束，AI 只看小图，不看完整平面图 |
| 9 | **鼠标分工** | 左键拖拽=框选，右键拖拽=旋转，中键/右键+Shift=平移 |
| 10 | **Phase 4 实现** | 纯确定性，只替换 Wall(10) 的面，不触碰 Open 面 |
