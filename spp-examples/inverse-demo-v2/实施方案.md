# SPP 弦粒子逆建模 Demo V2 — 实施方案

## 一、目标概述

基于 `spp-lib`（`spp-core.js` + `spp-inverse-engine.js`），实现全新的弦粒子逆重建 Demo：

1. **全屏 Canvas**：Three.js 铺满视口，所有交互在 3D 场景内完成
2. **多深度可视化**：粗网格 → 细化层叠加覆盖，按 depth 分层分色
3. **多选 + 递归识别**：Ctrl+Click / 框选多个 cell，批量 AI 细化
4. **房型图贴地板**：导入图片作为 3D 地面纹理，直观对比契合度

```

  ↓

  ↓

Phase2: AI 生成 faceOptions (Wall/Open)
  ↓

  ↓
用户 Ctrl+Click / 框选 → 选中若干 cell
  ↓

  ↓

  ↓

  ↓
快捷键 E → 导出 SPP JSON
```

## 四、核心模块设计

### 4.1 多层渲染器 (`renderer.js`)

按 `depth` 分组渲染，每层不同颜色 + 微微抬高：

| depth | 墙体颜色 | 地板颜色 | 墙高 | 抬高 Y |
|-------|---------|---------|------|--------|
| 0（粗网格） | `#d4886b` 砖红 | `#e8e8ef` 浅灰 | 2.8 | 0 |
| 1（一次细化）| `#6b8fd4` 蓝灰 | `#e0e8f0` 淡蓝 | 2.6 | 0.08 |
| 2（二次细化）| `#6bd49b` 青绿 | `#e0f0e8` 淡绿 | 2.4 | 0.16 |

已被细化覆盖的粗格子 → 墙体变半透明（`opacity: 0.15`），地板隐藏。

### 4.2 多选机制 (`selection.js`)

- **单击 cell**：选中/取消选中
- **Ctrl+Click**：追加多选
- **拖拽框选**：按住拖出矩形，框内 cell 全选
- **选中高亮**：蓝色描边 + 地板微微变亮
- 选中后底部出现 `"🔍 细化选中区域 (N cells)"` 按钮

### 4.3 房型图贴地板 (`floor-texture.js`)

- 用 `TextureLoader` 加载图片，UV 偏移到 crop 区域
- 铺在 `y=0.005`（低于所有网格地板）
- 透明度通过右下角滑块实时调节（0%~100%）

### 4.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| `T` | 俯视 / 透视切换 |
| `E` | 导出 SPP JSON（递归树结构） |
| `Escape` | 清空选择 |
| `Delete` | 移除选中 cell 的细化层（回退到上一层） |

## 五、spp-lib 需要的补丁

### 5.1 `flattenRecursiveCells` Bug 修复 + depth 传递

当前第 471 行 `gridSpan` 未定义。修复后增加 `depth` 参数：

```diff
- static flattenRecursiveCells(cells, parentWorldPos, parentWorldScale)
+ static flattenRecursiveCells(cells, parentWorldPos, parentWorldScale, depth = 0)
```

每个叶子节点输出中增加 `_depth` 字段，渲染器据此分层。

### 5.2 新增 `createBatchRefineContext(selectedCells, subGridSize)`

处理多选区域，计算最小外接矩形 + 提取边界约束，生成 AI Prompt 上下文。

### 5.3 边界约束验证

`integrateSubGrid` 后校验子网格边缘 faceOptions 是否与父级一致。

## 七、已确认的设计决策

| # | 决策项 | 结论 |
|---|--------|------|
| 1 | **Scale 策略** | 由 AI 根据区域复杂度自动决定（2、3 或 4），Prompt 中要求 AI 返回 `scale` 字段 |
| 2 | **递归深度上限** | **最多 4 层**（depth 0~3），超过后禁用细化按钮 |
| 3 | **导出格式** | **递归树结构 JSON**，保留 `subGrid` 嵌套，保留层级信息 |
| 4 | **Delete 回退** | 支持。选中已被细化的 cell → 按 `Delete` → 移除该 cell 的 `subGrid`，恢复为粗格子 |

### Delete 回退机制详细设计

```
用户选中 depth=1 的某些 cell
  ↓
按 Delete 键
  ↓
找到这些 cell 的父级 ParticleCell（depth=0）
  ↓
删除父级的 subGrid 属性
  ↓
父级 cell 恢复为不透明的粗格子
  ↓
Layer 1 中对应的细化 mesh 被移除
  ↓
重新 flatten + 渲染
```

> [!NOTE]
> 仅允许从最深层往上逐层回退。例如，depth=2 的 cell 必须先删除才能删除 depth=1。
